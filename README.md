# 🧠 Gabi Brain — 세컨드 브레인 시스템

이 볼트는 **수집 → 소화 → 생각 → 실행 → 회고**의 흐름으로 설계된 개인 지식 관리 시스템입니다.
옵시디언(Obsidian) + Git으로 운영되며, 작업일지는 GitHub 커밋 기반으로 자동 생성됩니다.

---

## 전체 구조

```
gabi_brain/
│
├── 00_Inbox/           ← 일단 던지기
├── 10_Capture/         ← 좋은 거 잡아두기
├── 20_Learn/           ← 공부한 것 정리
│   ├── courses/        ← 강의 노트
│   └── books/          ← 책/아티클 정리
├── 25_Home/            ← 집안 관리 (식재료, 소모품)
├── 30_Think/           ← 내 생각 발전시키기
│   ├── ideas/          ← 아이디어
│   └── patterns/       ← UI 패턴, 디자인 원칙
├── 40_Projects/        ← 실제로 만들기
├── 50_Strategy/        ← 큰 그림 그리기
├── 60_Journal/         ← 하루 돌아보기 (자동 생성)
├── 80_Archive/         ← 끝난 건 치우기
├── 90_Attachments/     ← 이미지/파일 자동저장
├── _templates/         ← Templater 템플릿
└── _system/            ← 시스템 설정, 스크립트
```

---

## 폴더별 설명

### 00_Inbox — 일단 던지기
**규칙: 분류 고민 없이 일단 여기에.**

아직 어디에 넣을지 모르는 것, 급하게 메모한 것, 나중에 정리할 것.
주 1회 리뷰 때 적절한 폴더로 이동시킵니다.

- ✅ 빠른 메모, 떠오른 생각, 임시 저장
- ❌ 여기에 오래 머물면 안 됨 (1주일 이상 방치 금지)

---

### 10_Capture — 좋은 거 잡아두기
**규칙: 웹클리퍼로 저장, 태그만 달고 끝.**

SNS에서 본 좋은 디자인, 블로그 글, 레퍼런스 스크랩.
내가 만든 게 아니라 **외부에서 가져온 것**.

```yaml
# 권장 프론트매터
---
type: capture
source: https://...
tags: [레퍼런스, 디자인, mobile-ui]
captured: 2026-03-22
---
```

- ✅ Dribbble, Behance, 블로그 글, 트위터 스크랩
- ❌ 내가 정리한 내용 (→ 20_Learn 또는 30_Think)

---

### 20_Learn — 공부한 것 정리
**규칙: 직접 수강/읽고 내 말로 정리한 것.**

강의를 보면서 정리한 노트, 책을 읽고 요약한 내용.
10_Capture와의 차이: **내가 소화한 흔적이 있는가**.

| 서브폴더 | 내용 |
|----------|------|
| `courses/` | 인프런, 유튜브, Udemy 등 강의 노트 |
| `books/` | 책, 긴 아티클 정리 |

```yaml
# 권장 프론트매터
---
type: learn
course: "Figma Auto Layout 마스터"
platform: 인프런
status: 진행중  # 진행중 / 완료
tags: [figma, auto-layout]
started: 2026-03-20
---
```

- ✅ 강의 요약, 단계별 정리, 실습 기록
- ❌ 단순 북마크 (→ 10_Capture)

---

### 30_Think — 내 생각 발전시키기
**규칙: 외부 자료가 아닌, 나만의 생각과 해석.**

Capture에서 영감을 받고, Learn에서 배운 것을 토대로
**내 관점에서 정리한 아이디어, 원칙, 패턴**.

| 서브폴더 | 내용 |
|----------|------|
| `ideas/` | 떠오른 아이디어, 사이드 프로젝트 구상 |
| `patterns/` | 반복되는 UI 패턴, 디자인 원칙 정리 |

```yaml
# 권장 프론트매터
---
type: think
tags: [onboarding, mobile, ux-pattern]
created: 2026-03-22
related:
  - "[[모바일 온보딩 레퍼런스]]"
  - "[[UX 심리학 강의 3강]]"
---
```

> 💡 **핵심**: 30_Think 노트에서 10_Capture, 20_Learn의 노트를 `[[링크]]`로 연결하세요.
> 이 연결이 세컨드 브레인의 진짜 가치를 만듭니다.

- ✅ "나는 이렇게 생각한다", 패턴 비교, 원칙 정립
- ❌ 남의 글 복붙 (→ 10_Capture)

---

### 40_Projects — 실제로 만들기
**규칙: 현재 진행 중이거나 곧 시작할 프로젝트.**

실제로 결과물을 만들어내는 것들. 마감이 있는 것.
프로젝트별 서브폴더를 만들고, 관련 노트를 링크합니다.

```
40_Projects/
├── design-system/
│   ├── README.md
│   └── 버튼-리뉴얼-진행노트.md
├── client-app/
└── portfolio/
```

> 💡 프로젝트 노트 상단에 관련 레퍼런스(10), 학습 노트(20), 아이디어(30)를 링크 섹션으로 넣어두세요.

- ✅ 진행 중인 작업, 진행 노트, 미팅 기록
- ❌ 끝난 프로젝트 (→ 80_Archive)

---

### 50_Strategy — 큰 그림 그리기
**규칙: 주차/월간/분기 단위 계획과 로드맵.**

마스터 플랜 HTML, 분기 목표, OKR 등 **장기적인 방향** 문서.
60_Journal의 자동 일지가 이 계획과 비교되어 진행률을 보여줍니다.

```
50_Strategy/
├── master-plan.html        ← 주차/마일스톤 기반 큰 계획
├── 2026-Q1-goals.md        ← 분기 목표
└── weekly-review.md         ← 주간 리뷰 템플릿
```

- ✅ 로드맵, 마일스톤, 분기 목표, 주간 리뷰
- ❌ 일일 할 일 (→ 60_Journal에서 자동 생성)

---

### 60_Journal — 하루 돌아보기
**규칙: 날짜별 자동 생성. 직접 쓸 필요 거의 없음.**

매일 밤 스크립트가 GitHub 커밋을 수집하고,
50_Strategy의 마스터 플랜과 비교하여 자동으로 작업일지를 생성합니다.

```
60_Journal/
├── 2026-03-20.md    ← 자동 생성
├── 2026-03-21.md
└── 2026-03-22.md
```

일지 내용 예시:
- 오늘 커밋 내역 (레포별)
- 마스터 플랜 매칭 (이번 주 계획 중 어떤 항목이 진행되었는지)
- 달성률

- ✅ 자동 생성 일지, 간단한 메모 추가 가능
- ❌ 수동으로 일일 TODO 작성 (불필요)

---

### 80_Archive — 끝난 건 치우기
**규칙: 완료된 프로젝트, 더 이상 안 보는 자료.**

삭제하는 게 아니라 **현재 동선에서 치우는 것**.
검색하면 여전히 나오지만, 일상적 탐색에서는 안 보입니다.

- ✅ 완료된 프로젝트, 오래된 레퍼런스, 끝난 강의
- ❌ 아직 참고할 수 있는 것

---

### 90_Attachments — 이미지/파일 자동저장
옵시디언 설정에서 첨부파일 기본 위치를 이 폴더로 지정하세요.
스크린샷, 이미지, PDF 등이 자동으로 여기 저장됩니다.

> 옵시디언 설정 > 파일 및 링크 > 첨부 파일 기본 위치 > `90_Attachments`

---

### _templates — Templater 템플릿
Templater 플러그인의 템플릿 폴더로 지정합니다.
새 노트를 만들 때 자동으로 구조화된 틀이 적용됩니다.

> 옵시디언 설정 > Templater > Template folder location > `_templates`

---

### _system — 시스템 파일
CLAUDE.md, 자동화 스크립트 등 노트가 아닌 시스템 파일.
언더스코어(`_`) 접두어로 정렬 시 맨 아래로 내려갑니다.

---

## 세컨드 브레인 흐름

```
외부 정보 ──→ 10_Capture (잡아두기)
                  │
                  ▼
             20_Learn (소화하기)
                  │
                  ▼
             30_Think (내 생각으로)  ←──── 영감
                  │
                  ▼
            40_Projects (실행)  ←── 50_Strategy (큰 방향)
                  │
                  ▼
            60_Journal (회고)  ──→  다음 날의 실행에 반영
                  │
                  ▼ (끝나면)
            80_Archive (보관)
```

**핵심 원칙:**
1. **폴더는 거주지일 뿐** — 진짜 연결은 `[[링크]]`와 태그로 만든다
2. **완벽한 분류보다 빠른 저장** — 모르겠으면 00_Inbox에 던진다
3. **주 1회 리뷰** — Inbox 비우기, Archive로 치우기, 링크 연결하기
