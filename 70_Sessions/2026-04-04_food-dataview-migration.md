## Dataview 마이그레이션

### 배경
- MD 테이블 방식은 항목 증가 시 필터링·태그·통계 불가
- Obsidian 볼트 환경 + Dataview 플러그인 설치됨
- JSX/React 불필요 — Dataview만으로 충분

### 선택한 구조
- 개별 항목 파일 방식: `20_Home/food/*.md` (YAML frontmatter)
- `household_Food.md`를 Dataview 대시보드로 전환

### 결과
- `20_Home/food/` 폴더에 **73개 개별 항목 파일** 생성
- YAML frontmatter 필드: `이름`, `카테고리`, `출처`, `가격`, `가격표시`, `용량`, `건강`, `건강메모`, `상태`, `메모`, `테스트목적`, `구매일`
- `상태` 값: `확정` / `테스트중` / `탈락` / `애매`
- 대시보드 쿼리: 카테고리별 테이블, 전체 현황 통계, 테스트중/탈락 자동 집계

### 새 항목 추가 방법
`20_Home/food/` 폴더에 아래 형식으로 파일 하나 생성하면 대시보드 자동 반영:

```yaml
---
이름: "[브랜드] 제품명"
카테고리: 간편식|밀키트|재료|반찬|음료|베이커리|파스타|양념|간식|주방용품
출처: 마켓컬리
가격: 6230
가격표시: "6,230원"
용량: 1인분
건강: ✅
건강메모: "메모"
상태: 확정
메모: "추가 메모"
---
```

## 시행착오

### 마켓컬리 URL 크롤링 불가
- `kurly.com` 및 `kurlink.kurly.com` 모두 403 반환
- → 스크린샷으로 제품 정보 직접 확인하는 방식으로 대체

### Dataview 구조 선택 고민
- 단일 파일 inline fields vs. 개별 파일 YAML frontmatter
- inline fields는 리스트 레벨 쿼리가 복잡함 → 개별 파일 방식 채택
- 개별 파일이 git diff도 명확하고 확장성 우수
